1 - Basic Concepts

1.1 - Values and Types

Hosket is a dynamically typed language. This means that variables do not have types; Only value can do that. There is no type definition in the language. All values have their own type.

All values in Hosket are first class values. This means that all values can be stored in variables, transmitted as arguments to other functions and returned as results.

There are only 4 types in Hosket: str, num, func and table. Since Hosket has no boolean or none type, numbers are substituted for it instead, such as: 0 stands for false condition, 1 stands for true condition and 0 also stands for none type. Sometimes str can also become a boolean when the empty string represents a false condition and there are text or data in the string that represents a true condition. Both tables and built-in types can become conditions.

The type num represents both integers and real (floating point) numbers, using two subtypes: integers and floats. Standard Hosket uses 64-bit integers and double-precision (64-bit) floats.

The str type represents an invariant byte string. Clean 8-bit Hosket: The string can contain any 8-bit value, including the embedded zero ('\0'). Hosket has nothing to do with encoding either; it makes no assumptions about the content of a string. The length of any string in Hosket must match the Hosket integer.

The type table implements an associative array and can access the table through 3 types: numbers, strings, and properties (or call it methods). Tables can be inconsistent; that is, they can contain values of any type. But sadly, because Hosket in this version has not been integrated to create additional entries for the table, so it can only be done through the built-in module table.

Tables are the only data structure mechanism in Hosket; they can be used to represent array, list, symbol, set, record, chart, regular tree, etc. To display the records, Hosket uses the field name as an index. The language supports this representation by providing a.name as syntax for a["name"]. There are several convenient ways to create tables in Hosket.

Like indexes, the values of the fields in the table can be of any type. Especially, since functions are first-class values, fields in tables can contain functions. So the tables can also carry the methods.

The indexing of tables does not comply with the definition of raw equality in language. The expressions a[i] and a[j] represent the same table component when and only when i and j are equal (i.e. equal when there is no metaphor). But if float numbers have integrated values equal to their respective integers (e.g. 1,0 == 1). But any float number that is used as a key, equal to an integer, will not be converted to that integer. For example, if you write a[2.0] = 1, the actual key only inserted into the table will be float 2.0 rather than integer 2.

The built-in function type returns a string describing the type of a given value.

1.2 - Operators

Hosket programming languages have all kinds of operators such as add, subtract, multiply, divide or other mathematics. For other operators like assignment operators or comparison operators, however, because Hosket does not have keywords and, or or not, it is replaced with operators such as &&, ||, !.

The mathematical operators that Hosket has are + (addition), - (subtraction), * (multiplication), / (division), % (modulus), ** (exponent) and // (floor division).

The comparison operators that Hosket has are == (equal), != (not equal), > (greater than), < (less than), >= (greater than or equal to), <= (less than or equals), && (and), || (or) and ! (not).

The assignment operators that Hosket has are =, +=, -=, *=, /=, %=, **= and //=.

1.3 - Error Handling

Some operations in Hosket may cause errors. An error interrupts the normal process of the program, which can continue by detecting the error.

The Hosket code can clearly signal errors by calling the error function. (This function never returns.)
